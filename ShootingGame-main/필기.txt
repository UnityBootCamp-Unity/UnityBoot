Unity 특수 폴더
1. Assets
: 유니티 프로젝트에서
사용하는 에셋이 포함되는 폴더

대부분의 API 함수는 에셋 폴더에
에셋이 존재한다라는 가정으로
경로 명시를 할 필요가 없음.

특수 폴더의 경우는 상황에 따라
추가해야 합니다.
2. Editor
해당 폴더에 존재하는 스크립트는
런타임 스크립트가 아닌
에디터 스크립트로 관리됩니다.

이 위치에 있는 스크립트가 Monobehaviour 상속이어도  게임 오브젝트에 할당 할 수 없습니다.

3. Plugins
유니티의 기능 확장을 위해 네이티브 DLL 파일(C/C++ 기반으로 작성되는 파일)을 넣는 경우가 있는데
해당 폴더 안에 넣어서 작업을 진행합니다.

4. Resources
리소스 폴더는 다운받을 리소스를 관리하는 폴더입니다. Resources.Load() 함수를 통해서 에셋을 로드해야 합니다.

5. StreamingAssets
  ios 등에서 비디오 파일 등을 액세스할때 사용되는 폴더
 Assets 폴더 바로 안 프로젝트에 배치되어야 함.


임포트 과정에서 무시되는 폴더
1. 숨김 처리되어있는 폴더
2. 확장자가 .tmp인 파일
3. .으로 시작하는 이름의 파일 또는 폴더
4. ~로 끝나는 파일 및 폴더
5. cvs 파일 및 폴더
